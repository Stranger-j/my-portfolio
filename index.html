  • Project: Our project aims to improve the experience of educators and students when having class via video-conferencing softwares. We want to create an interface with the following features: Students can easily signal the presenter with: raise hand, answer poll (yes/no, A/B/C/D) and reactions (emoji) from the audience (ex students can signal confusion/feeling lost).
  • My Role: UX designer and developer (Javascript)
  • Date: From Sept 2020 to Present

    • 1. Problem


  • First, through several brainstorming sessions, we defined problems that students are currently facing in a virtual educational environment.
  • We found that ‚ÄúRaising hand‚Äù in a meeting feature is hard for teachers to count quickly and hard for students to use (often buried behind other windows/drop downs) and If the class uses an external polling software, switching to this software disrupts the flow of the class.
  • And also we found that there are no features that allow students to express their feelings and emotions in a private, non-disruptive way (ex when a student is struggling to follow along).


    • 2. Task-based audience segments (mental model)


  • We defined the mental model for having an online class on Zoom to select groups of people whom we want to focus on to provide a tailored experience to users. First, we listed distinguishing tasks and grouped them where there is behavior affinity. At this stage, we would like to define actors. And we named the groups and wrote an description.
  • As far as presenters are concerned, presenters can be grouped by two core attributes: prior knowledge of video conferencing and degree in interest to engage people.
  • For participants, we group them into several categories based on two core attributes: participation and focus
  • In order to keep it manageable within the allotted time frame, we selected core groups to focus for this project. For presenters, we decided to target those with low and medium prior knowledge of video-conferencing tools , as well as those who look forhigh participation from their audience. For participants, we targeted both shy and active participants that are focused on the presentation.
